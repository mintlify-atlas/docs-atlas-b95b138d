Collections help you organize research papers into curated groups. Share collections with collaborators, manage papers efficiently, and keep your research organized.

## Overview

Collections are organized groups of papers that you can create, manage, and share with others. Each collection has metadata (title, description, tags) and contains any number of papers from arXiv.

## Creating Collections

<Steps>
  <Step title="Click Create Collection">
    Access the collections interface and click the "New Collection" button.
  </Step>
  
  <Step title="Enter details">
    Provide collection metadata:
    
    - **Title** (required): A clear, descriptive name
    - **Description**: What this collection is about
    - **Tags**: Comma-separated keywords for categorization
  </Step>
  
  <Step title="Save collection">
    Your collection is created instantly and appears in your collections list.
  </Step>
</Steps>

<CodeGroup>
```typescript Create Collection Request
POST /api/collections

{
  title: "Quantum Computing Fundamentals",
  description: "Essential papers for understanding quantum computing basics",
  tags: ["quantum", "computing", "fundamentals"]
}
```

```typescript Response
{
  _id: "507f1f77bcf86cd799439011",
  userId: "user123",
  title: "Quantum Computing Fundamentals",
  description: "Essential papers for understanding quantum computing basics",
  tags: ["quantum", "computing", "fundamentals"],
  createdAt: "2024-01-15T10:30:00Z",
  updatedAt: "2024-01-15T10:30:00Z",
  userRole: "owner",
  paperCount: 0
}
```
</CodeGroup>

## Managing Collections

### Viewing Collections

Your collections are displayed in two categories:

<Tabs>
  <Tab title="Recent Collections">
    The 4 most recently accessed collections appear at the top for quick access. Collections are sorted by last interaction time.
    
    ```typescript
    // From collection_state.svelte.ts:56-61
    get recentCollections() {
      return this.collections
        .filter((c) => c.lastInteraction)
        .sort((a, b) => new Date(b.lastInteraction).getTime() - 
                        new Date(a.lastInteraction).getTime())
        .slice(0, 4);
    }
    ```
  </Tab>
  
  <Tab title="All Collections">
    All other collections are shown below, sorted by creation date (newest first).
  </Tab>
</Tabs>

### Editing Collections

<Steps>
  <Step title="Open collection">
    Click on any collection to view its details.
  </Step>
  
  <Step title="Edit metadata">
    Update the title, description, or tags as needed.
  </Step>
  
  <Step title="Save changes">
    Changes are automatically synced to the server and reflected across all viewers.
  </Step>
</Steps>

<CodeGroup>
```typescript Update Collection
PATCH /api/collections/{collectionId}

{
  title: "Updated Collection Name",
  description: "New description",
  tags: ["new", "tags"]
}
```
</CodeGroup>

### Deleting Collections

<Warning>
  Deleting a collection is permanent and cannot be undone. All papers will be removed from the collection (but remain in arXiv).
</Warning>

<Steps>
  <Step title="Open collection options">
    Click the menu icon on the collection you want to delete.
  </Step>
  
  <Step title="Select delete">
    Choose "Delete Collection" from the options.
  </Step>
  
  <Step title="Confirm deletion">
    Confirm that you want to permanently delete the collection.
  </Step>
</Steps>

## Adding Papers to Collections

There are multiple ways to add papers to your collections:

### From Search Results

<Steps>
  <Step title="Find a paper">
    Search for papers and locate one you want to save.
  </Step>
  
  <Step title="Open collection menu">
    Click the "Add to Collection" button on the paper card.
  </Step>
  
  <Step title="Select collection">
    Choose which collection(s) to add the paper to.
  </Step>
</Steps>

### From Paper Details

When viewing a paper's full details, use the collection button to add it to one or more collections.

<CodeGroup>
```typescript Add Paper to Collection
POST /api/collections/{collectionId}/papers

{
  paperId: "2301.12345"
}
```

```typescript Remove Paper from Collection
DELETE /api/collections/{collectionId}/papers?paperId=2301.12345
```
</CodeGroup>

## Collection Roles & Permissions

Collections support three permission levels:

<Tabs>
  <Tab title="Owner">
    **Full Control**
    
    - Edit collection metadata
    - Add/remove papers
    - Manage members
    - Generate share links
    - Delete collection
  </Tab>
  
  <Tab title="Editor">
    **Collaborative Access**
    
    - Add/remove papers
    - Edit collection metadata
    - View all members
    - Cannot delete collection
    - Cannot manage permissions
  </Tab>
  
  <Tab title="Viewer">
    **Read-Only Access**
    
    - View collection contents
    - Browse papers in collection
    - Cannot modify anything
  </Tab>
</Tabs>

## Sharing Collections

Share your collections with others using secure share links:

<Steps>
  <Step title="Generate share link">
    Open collection settings and choose "Generate Share Link".
  </Step>
  
  <Step title="Select permission level">
    Choose whether to share as:
    - **Viewer**: Read-only access
    - **Editor**: Can add/remove papers and edit metadata
  </Step>
  
  <Step title="Share the link">
    Copy the link and send it to your collaborators.
  </Step>
</Steps>

<Info>
  Share links include a secure token that grants access without requiring the recipient to request permission.
</Info>

### Joining Shared Collections

When someone shares a collection with you:

<Steps>
  <Step title="Click the share link">
    Open the link they sent you.
  </Step>
  
  <Step title="Join collection">
    Click "Join Collection" to add it to your collections list.
  </Step>
  
  <Step title="Start collaborating">
    The collection appears in your collections with the appropriate permission level.
  </Step>
</Steps>

### Leaving Collections

To leave a shared collection:

<CodeGroup>
```typescript Leave Collection
DELETE /api/collections/{collectionId}/join
```
</CodeGroup>

The collection will be removed from your list, but other members can still access it.

## Collection Metadata

Each collection includes:

- **Basic Info**: Title, description, tags
- **Ownership**: User ID of creator, your permission level
- **Timestamps**: Created date, last updated, last interaction
- **Content**: Paper count, full paper list (when loaded)
- **Sharing**: Share link tokens for viewer and editor access

## Technical Implementation

Key features from `collection_state.svelte.ts`:

- **Smart caching**: Collections are cached for 5 minutes to reduce database calls
- **Optimistic updates**: UI updates immediately while syncing to server
- **Lazy loading**: Paper lists are only loaded when viewing a collection
- **Automatic timestamps**: `lastInteraction` updates whenever you view or modify a collection
